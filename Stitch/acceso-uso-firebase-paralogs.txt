# 🔥 Guía: Crear cuenta en Firebase y configurar base de datos en la nube

¡Excelente idea! Firebase es perfecto para almacenar logs, configuraciones, estados del bot, o datos de usuarios en la nube. Vamos paso a paso.

---

## 📋 Prerrequisitos

| Requisito | Estado |
|-----------|--------|
| ✅ Cuenta de Google (Gmail) | Necesaria |
| ✅ Navegador moderno (Chrome, Edge, Firefox) | Recomendado |
| ✅ Proyecto en Google Cloud (opcional, se crea automático) | Firebase lo gestiona |
| ✅ Tarjeta de crédito (solo para proyectos con pago, no obligatorio para inicio) | Spark Plan es gratis |

> 💡 **Firebase Spark Plan**: Gratis para siempre con límites generosos (perfecto para bots y proyectos pequeños).

## 🚀 Paso 1: Crear cuenta en Firebase

### 1.1 Ve a la consola de Firebase
Abre tu navegador y entra a:
```
https://console.firebase.google.com/
```

### 1.2 Inicia sesión con tu cuenta de Google
- Haz clic en **"Ir a la consola"**
- Selecciona tu cuenta de Google (o crea una nueva si no tienes)

### 1.3 Crea un nuevo proyecto
1.  Haz clic en **"Agregar proyecto"** (botón grande con `+`)
2.  **Nombre del proyecto**: `k8s-bot-db` (o el que prefieras)
3.  **ID del proyecto**: Se genera automático (ej: `k8s-bot-db-abc123`). ¡Anótalo!
4.  **Google Analytics**: Desactívalo por ahora (no es necesario para base de datos)
5.  Haz clic en **"Crear proyecto"**

⏱️ **Espera ~1-2 minutos** mientras Firebase configura tu proyecto.

### 1.4 Accede al proyecto
- Cuando termine, haz clic en **"Continuar"**
- Estarás en el dashboard de tu proyecto Firebase

---

## 🗄️ Paso 2: Configurar la base de datos (Firestore)

Recomiendo **Cloud Firestore** (más moderno y flexible que Realtime Database).

### 2.1 Activar Firestore
1.  En el menú lateral izquierdo, haz clic en **"Compilación"** → **"Firestore Database"**
2.  Haz clic en **"Crear base de datos"**
3.  **Modo de seguridad**: Selecciona **"Comenzar en modo de prueba"** (para desarrollo)
    > ⚠️ Esto permite lectura/escritura pública por 30 días. Luego configuraremos reglas reales.
4.  **Ubicación**: Elige una región cercana (ej: `eur3` para Europa, `nam5` para EE.UU.)
5.  Haz clic en **"Habilitar"**

⏱️ **Espera ~1 minuto** mientras se crea la base de datos.

### 2.2 Verificar que funciona
- Deberías ver una pantalla vacía con el mensaje: *"No hay documentos en esta colección"*
- ¡Listo! Tu base de datos en la nube está activa ✅

## 🔐 Paso 3: Obtener credenciales para conectar desde Python

### 3.1 Crear una cuenta de servicio (Service Account)

1.  En el dashboard de Firebase, haz clic en el **engranaje ⚙️** junto a "Visión general del proyecto"
2.  Ve a **"Cuentas de servicio"** → **"Generar nueva clave privada"**
3.  Selecciona **JSON** como formato
4.  Haz clic en **"Generar clave"**
5.  **Se descargará un archivo** como: `k8s-bot-db-firebase-adminsdk-abc123.json`

> ⚠️ **IMPORTANTE**: 
> - 🔒 **NUNCA compartas este archivo** ni lo subas a GitHub
> - 💾 Guárdalo en una carpeta segura: `C:\Users\pedrodulce\develop\antigravity\k8s-bot\firebase-key.json`
> - 📝 Añádelo a tu `.gitignore` si usas Git

### 3.2 (Opcional) Configurar variables de entorno

En tu archivo `.env` del bot, añade:

```env
# Firebase Configuration
FIREBASE_PROJECT_ID=k8s-bot-db-abc123
FIREBASE_CREDENTIALS_PATH=./firebase-key.json
FIREBASE_DATABASE_URL=https://k8s-bot-db-abc123.firebaseio.com
```

## 🐍 Paso 4: Conectar tu bot Python a Firebase

### 4.1 Instalar dependencias

```powershell
.\venv\Scripts\Activate.ps1
pip install firebase-admin
```

### 4.2 Crear módulo `firebase_client.py`

Crea un nuevo archivo `firebase_client.py` en tu carpeta `k8s-bot`:

```python
# firebase_client.py
# Cliente para conectar el bot a Firebase Firestore

import os
import firebase_admin
from firebase_admin import credentials, firestore
from pathlib import Path
from dotenv import load_dotenv

# Cargar variables de entorno
load_dotenv()

# Configuración
FIREBASE_CREDENTIALS = os.getenv("FIREBASE_CREDENTIALS_PATH", "./firebase-key.json")
FIREBASE_PROJECT = os.getenv("FIREBASE_PROJECT_ID")

# Inicializar Firebase (solo una vez)
def inicializar_firebase():
    """Inicializa la conexión con Firebase"""
    if not firebase_admin._apps:
        cred_path = Path(__file__).parent / FIREBASE_CREDENTIALS
        if not cred_path.exists():
            raise FileNotFoundError(f"❌ Credenciales de Firebase no encontradas: {cred_path}")
        
        cred = credentials.Certificate(str(cred_path))
        firebase_admin.initialize_app(cred, {
            'projectId': FIREBASE_PROJECT
        })
        print(f"✅ Firebase conectado al proyecto: {FIREBASE_PROJECT}")

# Obtener cliente Firestore
def get_firestore_client():
    """Devuelve el cliente de Firestore listo para usar"""
    inicializar_firebase()
    return firestore.client()

# ===========================================
# FUNCIONES DE EJEMPLO PARA EL BOT
# ===========================================

def guardar_log_bot(usuario_id, comando, respuesta, metadata=None):
    """Guarda un log de interacción del bot en Firestore"""
    try:
        db = get_firestore_client()
        
        doc_ref = db.collection('bot_logs').document()
        doc_ref.set({
            'usuario_id': usuario_id,
            'comando': comando,
            'respuesta': respuesta,
            'timestamp': firestore.SERVER_TIMESTAMP,
            'metadata': metadata or {}
        })
        return True
    except Exception as e:
        print(f"❌ Error guardando log: {e}")
        return False

def obtener_ultimo_estado_pod(namespace, pod_nombre):
    """Consulta el último estado registrado de un pod"""
    try:
        db = get_firestore_client()
        
        # Buscar logs del pod, ordenados por fecha
        logs = db.collection('pod_estados') \
            .where('namespace', '==', namespace) \
            .where('pod_nombre', '==', pod_nombre) \
            .order_by('timestamp', direction=firestore.Query.DESCENDING) \
            .limit(1) \
            .stream()
        
        for doc in logs:
            return doc.to_dict()
        return None
    except Exception as e:
        print(f"❌ Error consultando estado: {e}")
        return None

def guardar_estado_pod(namespace, pod_nombre, estado, metadata=None):
    """Registra el estado actual de un pod"""
    try:
        db = get_firestore_client()
        
        doc_ref = db.collection('pod_estados').document()
        doc_ref.set({
            'namespace': namespace,
            'pod_nombre': pod_nombre,
            'estado': estado,
            'timestamp': firestore.SERVER_TIMESTAMP,
            'metadata': metadata or {}
        })
        return True
    except Exception as e:
        print(f"❌ Error guardando estado: {e}")
        return False
```

## 🔐 Paso 7: Configurar reglas de seguridad (IMPORTANTE)

Después de los 30 días del "modo de prueba", debes configurar reglas reales.

### 7.1 Ve a Firestore → Reglas

### 7.2 Reglas básicas para desarrollo:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Logs del bot: solo escritura, lectura restringida
    match /bot_logs/{docId} {
      allow create: if true;  // Cualquiera puede escribir logs
      allow read, update, delete: if false;  // Nadie puede leer/eliminar (privado)
    }
    
    // Estados de pods: lectura pública, escritura solo desde backend
    match /pod_estados/{docId} {
      allow read: if true;
      allow write: if request.auth != null;  // Solo usuarios autenticados
    }
    
    // Alertas: lectura para usuarios específicos
    match /alertas_enviadas/{docId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null;
    }
    
    // Configuración: cada usuario solo accede a la suya
    match /configuracion/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

> 💡 **Para producción**: Implementa autenticación con Firebase Auth para identificar usuarios.

## 🚀 Paso 8: Integrar con tu bot de Telegram

En tu `bot.py`, importa y usa las funciones:

```python
# Al inicio de bot.py
from firebase_client import guardar_log_bot, guardar_estado_pod

# En el handler /start o cualquier comando:
async def start(update: Update, context: ContextTypes.DEFAULT_TYPE):
    usuario_id = str(update.effective_user.id)
    
    # Tu lógica existente...
    await update.message.reply_text("🤖 Bot K8s activo")
    
    # 👇 Guardar log en Firebase
    guardar_log_bot(
        usuario_id=usuario_id,
        comando="/start",
        respuesta="Bot iniciado",
        metadata={"chat_type": update.effective_chat.type}
    )
```
